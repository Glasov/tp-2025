# Лабораторная работа 4: Структурные и поведенческие паттерны проектирования

## Описание
Реализация 6 паттернов проектирования на Python 3.8:
- 3 поведенческих паттерна
- 3 структурных паттерна

## Паттерны

### 1. Стратегия (Strategy) - поведенческий
**Назначение:** Определяет семейство алгоритмов, инкапсулирует каждый из них и делает их взаимозаменяемыми.
**Пример:** `Sorter` с разными стратегиями сортировки (пузырьковая, быстрая, обратная).
**Файл:** `strategy.py`

### 2. Цепочка обязанностей (Chain of Responsibility) - поведенческий
**Назначение:** Позволяет передавать запросы последовательно по цепочке обработчиков.
**Пример:** Обработка запросов через цепочку `AuthHandler` → `PermissionHandler` → `ValidationHandler`.
**Файл:** `chain_of_responsibility.py`

### 3. Итератор (Iterator) - поведенческий
**Назначение:** Предоставляет способ последовательного доступа к элементам составного объекта.
**Пример:** `BookShelf` с итераторами для обхода книг (обычный, обратный, с фильтром).
**Файл:** `iterator.py`

### 4. Прокси (Proxy) - структурный
**Назначение:** Предоставляет объект-заместитель, который контролирует доступ к другому объекту.
**Пример:** `DatabaseProxy` с ленивой инициализацией, кэшированием и контролем доступа.
**Файл:** `proxy.py`

### 5. Мост (Bridge) - структурный
**Назначение:** Разделяет абстракцию и реализацию так, чтобы они могли изменяться независимо.
**Пример:** `RemoteControl` (абстракция) и `Device` (реализация) для управления разными устройствами.
**Файл:** `bridge.py`

### 6. Адаптер (Adapter) - структурный
**Назначение:** Позволяет объектам с несовместимыми интерфейсами работать вместе.
**Пример:** `UserSystemAdapter` для преобразования XML-интерфейса старой системы в JSON-интерфейс новой.
**Файл:** `adapter.py`
